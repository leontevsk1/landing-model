## Модель работы

- **Одна основная ветка:** `master`.

- **Фичи по желанию:** короткие ветки `feat/<тема>` только когда реально нужно параллельничать.

- ## 5 правил
1. **Коммить маленькими порциями.** Одна мысль — один коммит.

2. **Перед пушем — подтяни и примерь:**
   
   ```bash
   git fetch origin; git rebase origin/master
   ```

3. **Пушим только работающий код**.

4. **Не пушим мусор/артефакты** (видео/картинки → в `artifacts/`, игнорим).

5. **Если конфликт — решаем сами и сразу.** Нет — не пушим.

## Когда всё-таки ветка

- Если вдвоём трогаем один и тот же файл/модуль.

- Имя: `feat/uav-landing` или `fix/rss-noise`.

- Слияние обратно: **rebase + fast-forward** (без PR).
  
  ```bash
  git switch master
  gsync
  git merge --ff-only feat/uav-landing
  git branch -D feat/uav-landing
  git push
  ```

## Экстренные случаи

- **Откат последнего пуша (мягко):**
  
  ```bash
  git revert HEAD
  git push
  ```

- **Сделать снимок перед рискованным рефакторингом:**
  
  ```bash
  git tag -a v0-snapshot -m "перед рефакторингом"; git push origin v0-snapshot
  ```
